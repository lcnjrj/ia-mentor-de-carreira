## Visão do Dia a Dia

Como é o trabalho de uma Desenvolvedora Back-end:

* 
**Modelagem de Regras de Negócio:** Traduzir problemas complexos em algoritmos lógicos usando Python. 


* 
**Desenvolvimento de APIs:** Construir os sistemas que processam dados e os entregam de forma organizada para o front-end ou outros serviços. 


* 
**Integração de Modelos de IA:** Conectar aplicações a APIs de inteligência artificial (como Gemini ou OpenAI) para adicionar camadas de inteligência ao produto. 


* 
**Gerenciamento de Persistência:** Estruturar como as informações são salvas e buscadas em bancos de dados, garantindo integridade e performance. 


* 
**Automação e Deploy:** Utilizar sua base em Shell Script para automatizar o ambiente de desenvolvimento e garantir que o código rode perfeitamente em servidores Linux.

## Roadmap de 90 Dias

**Adaptado para:** 20 horas/semana 

### Mês 1 - Fundamentos e Estruturação Profissional

* 
**Semana 1-2:** Python Moderno: Focar em decoradores, geradores e Programação Orientada a Objetos (POO). Ambientes Virtuais: Domínio de venv, pip e gestão de dependências. 


* 
**Semana 3-4:** Banco de Dados: SQL do zero ao avançado. Joins, Indexação e normalização de tabelas. Git Avançado: Aprender a documentar repositórios de forma que recrutadores entendam seu processo de raciocínio. 



### Mês 2 - Desenvolvimento de APIs e IA

* 
**Semana 5-6:** FastAPI: Criação de rotas, validação de dados com Pydantic e documentação automática com Swagger. Integração de IA: Criar scripts Python que interagem com a API do Gemini para processar grandes volumes de texto. 


* 
**Semana 7-8:** ORM (SQLAlchemy/SQLModel): Manipular bancos de dados usando Python em vez de SQL puro. Automação de Testes: Escrever testes unitários com Pytest para garantir que seu código seja robusto. 



### Mês 3 - Projeto Final e Mercado

* 
**Semana 9-10:** Desenvolvimento do Portfólio: Focar no projeto "Symphony AI" (veja abaixo). Docker: Empacotar sua aplicação Python para que ela possa ser instalada em qualquer servidor com um comando. 


* 
**Semana 11-12:** Revisão de Portfólio: Ajustar READMEs e LinkedIn. Simulação de Entrevistas: Focar na transferência de habilidades (como seus 20 anos de Linux resolvem problemas de hoje).

## Projeto de Portfólio

**Projeto:** Smart Knowledge Organizer (SKO) 

* 
**O que fazer:** Desenvolver uma API de backend que receba grandes quantidades de informações desorganizadas (textos, listas de livros, catálogos de mídia) e use IA para categorizar, extrair metadados e organizar tudo em um banco de dados estruturado, permitindo buscas inteligentes. 


* **Entregáveis:**
* API com rotas para upload de dados e consulta filtrada. 


* Integração funcional com a API do Gemini para classificação automática. 


* Script de instalação automática em Bash que configura todo o ambiente (Python + Banco). 




* **Critérios de Aceitação:**
* Documentação técnica impecável (Wiki no GitHub). 


* Cobertura de testes superior a 80%. 


* Código seguindo as normas da PEP 8. 





> Dica: Use seu interesse em taxonomia para criar um sistema de tags inteligente que aprenda com os dados inseridos.

## Roteiro de Entrevistas

* 
**Pergunta 1:** Como sua vasta experiência como usuária avançada de Linux influencia sua forma de escrever código hoje? 


* 
**Resposta:** Destaque que sua familiaridade com o sistema operacional permite entender o que acontece "debaixo do capô" (memória, processos, permissões), o que resulta em um código backend mais eficiente e fácil de depurar em ambientes de produção. 




* 
**Pergunta 2:** Por que você escolheu transitar para o desenvolvimento agora e como lida com o aprendizado de novas tecnologias? 


* 
**Resposta:** Enfatize que você nunca parou de aprender (citando sua graduação em ADS) e que sua maturidade traz uma habilidade de resolução de problemas e organização que iniciantes mais jovens costumam não ter. 




* 
**Pergunta 3:** Explique como você integraria uma IA em um fluxo de trabalho de backend existente. 


* 
**Resposta:** Fale sobre o uso de APIs para tarefas que antes eram impossíveis de programar "na mão", como análise de sentimento ou resumo de documentos, sempre tratando as respostas da IA como dados que precisam de validação. 




* 
**Pergunta 4:** Como você garante a segurança de uma API Python? 


* 
**Resposta:** Mencione o uso de variáveis de ambiente para chaves secretas, validação rigorosa de entrada de dados para evitar SQL Injection e o uso de autenticação via Tokens (JWT). 




* 
**Pergunta 5:** Como você organiza seu fluxo de trabalho quando recebe uma tarefa complexa? 


* **Resposta:** Fale sobre sua facilidade em pesquisa e organização. Explique que você primeiro quebra o problema em partes menores, documenta a lógica e depois implementa cada módulo, testando individualmente.
